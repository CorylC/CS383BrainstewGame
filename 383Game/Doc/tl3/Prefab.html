Bullet Prefab Documentation

Description

This prefab represents the bullet object used in the player's weapon system. It handles movement, hit detection, and visual effects when a bullet is fired. Any updates to this prefab automatically apply across all weapon systems using it.

Components

Bullet (Script)

  • Speed: Controls how fast the bullet travels (default: 15).
  • Damage: The amount of health deducted from the enemy on hit (default: 50).
  • Lifetime: How long the bullet exists before despawning (default: 0.2 seconds).

Sprite Renderer

  • Displays the bullet sprite Bullet_2_2_0.
  • Can be customized to change the bullet's appearance.

Rigidbody2D

  • Body Type: Dynamic.
  • Gravity Scale: 0 (no gravity effect).
  • Enables the bullet to move via physics without falling.

Capsule Collider 2D

  • Defines the collision boundary of the bullet.
  • Used for detecting collisions with enemies or obstacles.

Prefab Goals In Summary

  • Consistency: Ensures bullet behavior is uniform across all weapons.
  • Easy Updates: Changes to the prefab auto-apply where used.
  • Reusability: Drop into other weapons, scenes, or projects easily.

Common Questions and Answers

Q: How do I change bullet visuals?
A: Swap the sprite inside the Sprite Renderer or adjust its color and material settings.

Q: How do I make the bullet last longer?
A: Increase the Lifetime value in the Bullet (Script).

Q: Can the bullet pierce enemies?
A: Yes, by adjusting logic inside the Bullet script to allow multiple hits before destruction.
